file naming conventions: all documentation files should be named in a clear and consistent manner
 - use YYYYMMMDD format for dates (e.g., `20260326_development_plan.md`)
 - start with a descriptive title that reflects the content of the document
 - using lowercase letters and underscores to separate words (e.g., `api_documentation.md`, `user_guide.md`).
 - file type: all documentation files should be in markdown format (.md) to ensure compatibility and ease of use across different platforms and tools.
 - content guidelines: documentation should be clear, concise, and well-organized. It should include relevant information, instructions, and examples to help users understand and utilize the project effectively. Use headings, bullet points, and code snippets where appropriate to enhance readability and comprehension.
 - version control: all documentation files should be maintained in a version control system (e.g., Git) to track changes, collaborate with team members, and ensure that the documentation is up-to-date with the latest developments in the project.
